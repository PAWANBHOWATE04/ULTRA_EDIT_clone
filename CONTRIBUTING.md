# Contributing

Thanks for your interest in contributing to ULTRA_EDIT_clone — your help is appreciated!

Quickstart
1. Fork this repository.
2. Create a branch for your change:
   - `git checkout -b feat/short-description` (features)
   - `git checkout -b fix/short-description` (bugfixes)
3. Make changes, run any local checks, and write tests where applicable.
4. Commit with a clear message (Conventional Commits style recommended, e.g. `feat: add toolbar icons`).
5. Push to your fork and open a Pull Request against `main`.

Pull request checklist
- [ ] The PR has a clear title and description of what changed and why.
- [ ] Changes are limited in scope and small enough to review.
- [ ] Files are formatted consistently (follow existing style in `style.css`).
- [ ] If visual changes were made, include screenshots/GIFs in the PR description.
- [ ] CI checks (if any) pass.

Code style
- HTML: follow existing structure and semantic tags.
- CSS: follow the project's indentation and naming style. Keep selectors specific and avoid unnecessary global styles.

Labels & Issues
- Use labels like `good-first-issue` and `help wanted` for beginner-friendly tasks.
- When filing an issue, include steps to reproduce, expected vs actual behavior, and screenshots if relevant.

Reporting security issues
- Do not create a public issue for security vulnerabilities. Contact the maintainers directly via the repository owner’s GitHub profile (https://github.com/PAWANBHOWATE04) to report sensitive issues.

Communication
- Be respectful and follow the project's Code of Conduct.
- Maintainers will review PRs as time permits — please be patient.

Thanks — looking forward to your contributions!
